# 云图 · 产品设计说明

> 本文档描述云图各页面的功能结构和交互逻辑。
> 具体视觉参数（颜色、字号、间距）由设计师根据设计规范填写。

---

## 产品定位

移动端 web app，移动端是主力，桌面端是扩展。
所有页面以手机屏幕为基准设计，桌面端在此基础上适配。

---

## 设计原则

**内容优先**
头部和底部是工具，不是主角。用户打开页面，第一眼应该落在内容上。

**沉浸感**
导航尽可能轻，不抢内容的戏。参考小红书的底部 Tab Bar 模式，顶部空间全部让给内容。

**克制**
不做不必要的动效、装饰、弹窗。每一个元素存在都有理由。

**中文友好**
排版、断行、标点，都要适配中文阅读习惯。

## 页面总览

```
/                    首页

/tools/[slug]        产品详情页

```

## 全局导航

### 移动端：底部 Tab Bar

```
[首页]     [发现]     [关于]
```

三个 tab，固定在屏幕底部。底部 Tab Bar 是移动端唯一的主导航，顶部不放导航。

「关于」tab 包含：云图介绍、联系方式。不做用户系统，不叫「我的」。

### 二级页顶部栏

只在详情页等二级页出现，显示返回箭头和当前页名称。首页、列表页、关于页不需要顶部栏。

### 桌面端

顶部导航栏，滚动后出现背景。底部 Tab Bar 隐藏。

### 页面结构（从上到下）

**顶部标识**
仅展示「云图」文字 logo，轻量，不做导航功能。无搜索入口。

**Hero 区**
一句话说清楚云图是什么，配一个「浏览所有工具」的跳转按钮。文字为主，不做图片背景。

**本周编辑精选**
编辑人工挑选，最多 3 个，纵向排列。每个工具一张大卡片，全宽展示。卡片需要有「编辑精选」的标识。整张卡片可点击，跳转详情页。

**最新上架**
最新发布的工具，两列网格排列。每个工具一张小卡片，显示名称、简短描述、场景标签、点赞数。整张卡片可点击。底部有「查看全部」入口。

**底部 Tab Bar**

### 交互说明

- 精选卡片和最新卡片都是整张可点击，不只是按钮
- 点赞只在列表和详情页操作，首页卡片只展示点赞数，不可点击

## 2. 产品列表页 `/tools`

### 功能目标

让用户快速找到符合自己场景的工具。发现和筛选是核心。

### 线框图

```
┌───────────────────────────────────┐
│  全部  设计  写作  编程  效率 ›   │  ← 标签筛选栏，吸顶，横向滚动
├───────────────────────────────────┤
│  最新上架          最多点赞       │  ← 排序切换
├───────────────────────────────────┤
│                                   │
│  [图标]  工具名称          ♥ 128  │  ← 列表行，全宽
│          一句话描述，最多两行      │
│          [标签]  开发者：xxx       │
│  ─────────────────────────────   │
│  [图标]  工具名称          ♥ 64   │
│          一句话描述                │
│          [标签]  开发者：xxx       │
│  ─────────────────────────────   │
│  [图标]  工具名称          ♥ 32   │
│          ...                      │
│                                   │
├───────────────────────────────────┤
│  [🏠 首页]  [🔍 发现]  [💡 关于]  │
└───────────────────────────────────┘
```

### 页面结构（从上到下）

**场景标签筛选栏**
吸顶，横向滚动。标签选项：全部、设计、写作、编程、效率、图像、音视频、其他。默认「全部」。单选。

**排序切换**
两个选项：最新上架、最多点赞。

**工具列表**
全宽列表行，每行显示：工具图标、工具名称、一句话描述、场景标签、开发者名称、点赞数。整行可点击跳转详情。点赞数可以单独点击触发点赞。

**底部 Tab Bar**

### 交互说明

- 标签筛选栏吸顶，用户滚动时始终可见
- 点赞触发时有视觉反馈，防止重复点击
- 无搜索功能（MVP 不做）
- 无分页按钮，向下滚动加载更多

### 页面结构（从上到下）

**顶部栏**
返回箭头 + 工具名称，吸顶。

**工具信息区**
工具图标（大）、工具名称、一句话描述、场景标签、开发者信息、发布日期。

**评测正文**
MDX 渲染。固定结构：
- 它解决什么问题
- 谁该用
- 谁不该用
- 和同类产品比
- 定价
- 编辑一句话判断（用引用块样式突出显示）

正文图片全宽展示，突破页面内边距出血到两侧。

**相似工具**
横向滚动，推荐同场景标签的其他工具。每个显示图标和名称，可点击跳转。

**用户反馈**
显示已有反馈列表，时间降序排列。超过三行的反馈内容折叠，点击展开。底部有输入框，点击后展开为多行，可提交新反馈。昵称选填，内容必填。

**底部悬浮 CTA 栏**
固定在屏幕底部，替代 Tab Bar。左侧：点赞按钮和计数。右侧：「立即试用」主操作按钮，点击跳转工具官网。详情页不显示 Tab Bar。

### 交互说明

- 底部留足空间，防止正文内容被悬浮 CTA 遮挡
- 输入反馈时键盘弹起，页面上推，输入框保持可见
- 悬浮 CTA 是此页最重要的转化入口，始终可见

## 桌面端扩展

以下是移动端和桌面端的主要差异，其余保持一致：

| 元素 | 移动端 | 桌面端 |
|---|---|---|
| 主导航 | 底部 Tab Bar | 顶部 Navbar |
| 首页精选卡片 | 单列全宽 | 三列横排 |
| 首页最新上架 | 两列 | 四列 |
| 详情页布局 | 单列，无侧边栏 | 左侧正文 + 右侧边栏 |
| 详情页 CTA | 底部悬浮栏 | 右侧边栏内固定按钮 |
| 关于页服务介绍 | 纵向列表 | 三栏卡片 |

## 组件清单

| 组件 | 用途 |
|---|---|
| `TabBar` | 底部主导航，移动端 |
| `PageHeader` | 二级页顶部返回栏 |
| `Navbar` | 顶部导航，桌面端 |
| `ToolCardLarge` | 精选大卡片，首页用 |
| `ToolCardSmall` | 两列小卡片，首页最新用 |
| `ToolListItem` | 列表行，产品列表页用 |
| `TagBadge` | 场景标签 |
| `TagFilter` | 标签筛选栏，横向滚动，吸顶 |
| `SortToggle` | 排序切换 |
| `LikeButton` | 点赞按钮，含计数和反馈动效 |
| `FixedCTA` | 底部悬浮 CTA，详情页专用 |
| `MDXRenderer` | 评测正文渲染，含图片出血 |
| `HorizontalScroll` | 横向滚动容器，相似工具用 |
| `FeedbackList` | 反馈列表，超长折叠 |
| `FeedbackForm` | 反馈输入框，点击展开 |
