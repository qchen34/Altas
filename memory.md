## 核心定位

**一句话**：为中文 AI 用户提供"发现 + 试用 + 交流"的平台。

**精准服务三类人**：
1. Indie 开发者 / Vibe Coder（供给侧）
2. 普通用户 / 解决具体问题的人（需求侧）
3. 高密度信息消费者 / 投资人 / 技术迷（放大器）

**形态参照**：
- 内容深度 → 少数派
- 发现体验 → 豌豆荚
- 开发者关系 → ProductHunt
- 终局形态 → Steam + App Store

**云图不是**：
- 不是爬虫目录站（Toolify 那种）
- 不是通用 AI 聚合平台（Poe 那种）
- 不是 API 中转站（虽然可以合作，但不是主业）
- 不是纯内容媒体（少数派那种，商业模式太弱）

## 三类用户的核心痛点（已验证）

### 用户 A · Indie 开发者 / Vibe Coder

**最核心的痛点**（按重要性排序）：
1. 做出来没人看——缺曝光场
2. 拿不到真实的用户反馈——不知道卡在哪
3. 无法快速试错迭代——没有实验场
4. 建立不了个人品牌——作品集散落各处
5. 商业化路径不清晰——不知道从哪收钱

**关键洞察**：vibe coder 需要的不是"展示台"，是"实验场"——每次迭代都要看数据对比。

### 用户 B · 普通用户

**最核心的痛点**：
1. 有具体场景需求，但不知道哪个工具最合适（比如：品牌设计大学生要做样机看板）
2. 工具太多，筛选成本极高（Toolify 28000 个工具，无从下手）
3. 试错成本高（要翻墙 / 要英文 / 要美元付费才能试用）
4. 信息不对称（功能分类对普通人没用，场景分类才有用）

**关键洞察**：用户需要的是"场景匹配"，不是"功能搜索"。

### 用户 C · 高密度信息消费者

**最核心的痛点**：
1. 缺一个有判断力的中文信息源（English-only 信息源门槛高）
2. 不想被 SEO 垃圾内容浪费时间
3. 想找到投资 / 合作 / 学习机会

**关键洞察**：这批人是云图的放大器，他们不付钱但他们转发。要让他们觉得转发云图是在"展示品味"。

## 商业模式（四阶段）

### Phase 0（0-12 个月）· 养活自己

**主要收入**：AI API 中转站渠道返佣（10-30%）
**逻辑**：导流不经营，法律风险低，启动门槛低
**目标**：月收入 3000-10000 元，覆盖服务器成本

### Phase 1（12-24 个月）· 验证变现

**主要收入**：开发者首发推广费（$299-999/次）
**辅助收入**：Pro 用户订阅（¥39/月 或 ¥299/年）
**前提条件**：月活达到 5-10 万
**目标**：MRR $5000-15000

### Phase 2（24-36 个月）· 支付撮合

**主要收入**：集成支付 SDK，抽 10-15% 撮合费
**参照**：Substack 10%，Patreon 8%，App Store 30%
**目标**：月度 GMV $100K-1M

### Phase 3（36 个月+）· App Store 终局

完整的 AI App Store：内嵌试用 + 合规副本服务 + 支付通道 + 开发者后台
届时考虑融资、团队、合规升级

## 已确定的关键决策

### 做什么

- [x] 中文市场优先（不是出海发行，是服务中文用户）
- [x] 内容和编辑判断驱动（不是爬虫/SEO 驱动）
- [x] 按场景分类（不是按功能分类）
- [x] 先聚焦开发者侧（供给侧先行）
- [x] 海外部署起步（Vercel/Cloudflare，不做 ICP 备案）
- [x] Phase 0 用中转站返佣做初始现金流
- [x] 内嵌试用是核心产品差异点（用户不用跳出平台）
- [x] AI 辅助内容生产（AI 初稿 + 人工润色）

### 不做什么（已排除）

- [ ] ~~自己做 AI Token 中转站~~（红海，账号池风险高，主业分散）
- [ ] ~~Token 质押 / 回收业务~~（中国境内金融违规，接近非法集资）
- [ ] ~~早期上链发 Token~~（过早发币会让项目变成"币圈项目"，吓跑 AI 圈用户）
- [ ] ~~强制实名注册~~（增加合规义务，早期不必要）
- [ ] ~~做通用 AI 目录大而全~~（Toolify 做了，竞争没有优势）
- [ ] ~~出海发行平台定位~~（中文 indie 开发者真正痛点不在出海，在发现）
- [ ] ~~做内容站变成少数派~~（商业模式天花板太低）
- [ ] ~~做社区评论功能（早期）~~（产生舆论属性，增加合规风险）

## 竞品认知

| 竞品 | 类型 | 弱点 | 云图的差异 |
|---|---|---|---|
| Toolify | AI 工具目录 | 爬虫生成，无编辑判断，按功能分类 | 编辑精选，按场景分类 |
| 少数派 | 内容平台 | 无试用闭环，开发者工具弱，商业模式单一 | 试用闭环，开发者后台，多层变现 |
| ProductHunt | 发现平台 | 全英文，不服务中文市场，质量越来越水 | 中文，垂直 AI，编辑把关 |
| Lovart 案例 | 参考案例 | 海外版无法在国内合规服务（Lovart → 星流） | 说明了中国 indie 需要合规基础设施（Phase 3 方向） |
| GPT Store | 平台内 App Store | 只限 OpenAI 生态，不含 Web App | 跨模型，覆盖所有 indie web app |

## 灰色地带原则（Phase 0）

**可以做的灰色**：
- 海外部署 + 不做 ICP 备案（早期流量小，执法不会关注）
- 通过中转站调用境外大模型（责任在中转站，不在云图）
- 个人收款（月流水 < 50 万人民币不触发税务关注）
- 不主动声明调用哪个模型（模糊处理 = 保护）

**绝对不能踩的红线**：
- 政治 / 色情 / 未成年人相关内容
- 声称用 A 模型（已备案），实际用 B 模型（未备案）→ 欺诈
- Token 质押承诺收益 → 非法集资风险
- 境内发行加密货币 → 2021 年九部委公告明确违法

**合规升级触发点**（到了这些节点才考虑合规）：
- MAU > 10 万 → 考虑 ICP 备案
- MRR > $10000/月 → 注册公司
- 用户问"你们正规吗" → 做合规公示
- 收到第一个监管问询 → 立即合规化

## 产品形态关键设计原则

1. **内嵌试用 > 跳转** · 用户不离开云图就能试用工具，这是核心竞争力
2. **场景分类 > 功能分类** · "品牌设计大学生要做样机"比"图像生成"更有用
3. **开发者数据看板是黏性来源** · 让开发者每天登录看数据，不是发完就走
4. **中转返佣模块不超过页面 5% 视觉占比** · 配菜不能盖过主菜
5. **v1/v2/v3 版本迭代记录** · 云图要成为 vibe coder 的"实验记录本"
6. **不做站内社区评论（早期）** · 用微信群 / 即刻 / 小红书替代，避免舆论属性
